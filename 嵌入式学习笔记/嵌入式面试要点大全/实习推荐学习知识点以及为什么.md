从git开始   对嵌入式软件来说，你们可能可能习惯了单打独斗，stm32或者esp32，或者什么
ti板，再什么ardunio之流就不说了，基本上学校里面的项目都是个人的项目，一个人不管是抄也好自己写也好，完成了对某传感器的驱动，写对屏幕的处理函数
，联网上云蓝牙，控制小车，舵机等等完成项目。
什么平衡车，蓝牙遥控舵机蜘蛛，循迹车来说，这些个项目都很小，不需要什么版本管理，也不需要多人协同。
但是在正常的工作过程中，多人协同完成任务怎么办呢，一人一份代码，你把这里改了我把那块改了，那最后的项目怎么交付呢，你突然发现上一个版本的代码稳定性
比现在的代码稳定性好的一倍，但是你改代码改的面目全非了，怎么回退保存版本


这个答案的解法 git

git可以说是一个项目版本工具，他有主分支，master ，以及其他的，自带版本管理系统，你必须得知道一些关于这个的常识  指令在git文件

那么首先git clone 一个ssh链接   git clone 102.23.2.3@xxx.git   这是ssh类型的
git clone  htpp/sss/sss       这是http链接  
比方说这是你们公司的代码 那么你通过git clone指令就可以把他们下载到本地
